03 / 11
The full lifecycle, with iterations
Real events from a real run. Three actors emit them: you, the agent, the grader. The loop continues until the grader returns satisfied or the iteration cap is hit.
user.define_outcome user
You attach a rubric to the session. The agent starts working immediately on receipt. No additional message event needed.
Iteration 0
agent.message · agent.tool_use agent
Agent plans, reads files, runs scripts, produces a first draft of the deliverable.
span.outcome_evaluation_start grader · iter 0
Grader spins up in its own context window. Reads rubric and the agent's output. Cannot see the agent's reasoning.
span.outcome_evaluation_ongoing heartbeat
Periodic heartbeat while the grader evaluates. Grader's internal reasoning is opaque.
span.outcome_evaluation_end result: needs_revision
Grader returns per-criterion feedback: which checks pass, which fail, what to change. Agent reads the explanation and starts a new iteration.
Iteration 1
agent.message · agent.tool_use agent
Agent revises in response to the specific gaps the grader identified. Edits files, reruns scripts, updates the deliverable.
span.outcome_evaluation_start grader · iter 1
Fresh grader instance. Re-evaluates the new output against the same rubric. The score does not drift across iterations because the grader has no memory of the previous run.
span.outcome_evaluation_end result: satisfied
All criteria met. Grader returns explanation listing which checks passed.
session.idle deliverables ready
Files are in /mnt/session/outputs/ on the agent's container. Pull them via the Files API scoped to the session.
Default cap is 3 iterations. Maximum is 20. The session retains history across iterations so the agent reads prior feedback, but each grader is fresh.
